## 🔴 致命问题 (必须修复)

### 1. db.py 上帝文件 (架构缺陷)
- **问题**: 913行/50方法混合8种职责
- **风险**: 无法维护，无法测试
- **建议**: 拆分为独立的 Repository 类

### 2. 无 Pydantic 模型
- **问题**: API 无类型安全，无请求/响应验证
- **风险**: 运行时错误，无法自动文档
- **建议**: 为每个 API 添加 Pydantic Model

### 3. Session 内存泄漏
- **问题**: `_sessions: Dict[str, dict] = {}` 无过期清理
- **风险**: 内存持续增长，最终 OOM
- **建议**: 添加定时清理过期 session

### 4. 无数据库迁移机制
- **问题**: schema 变更直接修改表结构
- **风险**: 生产环境升级困难
- **建议**: 引入 alembic 或自建迁移
