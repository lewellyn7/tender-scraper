## 三、任务详情

### 🔴 P0-1：测试 ERROR 修复 + 覆盖提升

**问题：**
```
ERROR tests/test_harvest/test_anti_detect.py
ERROR tests/test_harvest/test_human_behavior_engine.py
ERROR tests/test_harvest/test_integration.py
ERROR tests/test_harvest/test_security_utils.py
```

**根因（预判）：**
- `test_harvest/` 测试文件缺少正确路径映射（相对于项目 root 的 sys.path）
- `conftest.py` 仅 38 LOC，fixture 不完整
- 某些测试依赖的模块 import 路径问题

**修复步骤：**
1. 诊断具体 import 错误：`cd ~/tender-scraper && source venv/bin/activate && python -m pytest tests/test_harvest/test_anti_detect.py --collect-only 2>&1`
2. 修复 `conftest.py` 添加 `sys.path.insert` + 完整 fixture
3. 确认 4 个 ERROR → 0 ERROR
4. 补充缺失的 API 端到端测试
5. 补充数据库迁移测试

**验收标准：** `pytest tests/ --collect-only` 显示 75+ tests collected, 0 errors

---

### 🔴 P0-2：N+1 查询优化

**问题：** `routes/projects.py` 中每个项目 2 次独立 SQL（is_favorite + annotation），分页 20 条 = 40 次查询

**优化方案（来自 API优化报告）：**
- 改为 `WHERE project_url IN (...)` 批量查询，2 次替代 40 次
- 预期延迟改善：-80%

**涉及文件：**
- `app/api/routes/projects.py`
- `app/database/db.py`

**验收标准：** 分页页面加载数据库查询 ≤ 5 次（当前 40+ 次）

---

### 🟠 P1-1：日志标准化

**问题：**
- loguru 已安装，但各模块使用方式不统一（有的带 extra，有的直接 log.info）
- 无统一格式模板（request_id、user_id 缺失）
- 日志级别不一致

**解决方案：**
- 在 `app/utils/` 或 `app/core/` 创建 `logging_config.py`
- 统一格式：`[<time>] <level> | <module> | <message> | extra={...}`
- 要求所有模块使用统一 logger 实例

**验收标准：** 全局日志格式一致，包含 request_id/trace_id

---

### 🟠 P1-2：错误处理增强

**问题：**
- routes 层部分 try/except pass，部分无异常处理
- 全局异常未统一捕获
- 异常响应格式不统一

**解决方案：**
- 在 `app/middleware/` 添加 `exception_handler_middleware.py`
- 统一异常响应格式：`{"error": "...", "code": "...", "detail": "..."}`
- 覆盖 4xx/5xx，区分业务异常 vs 系统异常

**验收标准：** 所有 API 路由异常返回统一 JSON 格式，不泄漏内部堆栈

---

### 🟡 P2：CI/CD 优化

**前置条件：** 测试 ERROR 修复完成（否则 CI 跑不通）

**优化内容：**
1. 确认 `pytest.yml` GitHub Actions 能正常运行（测试 ERROR 修复后）
2. 添加 pytest-cov 集成，CI 报告覆盖率
3. 添加ruff lint 检查 step
4. 优化缓存策略（pip cache、venv cache）

**验收标准：** GitHub Actions 运行成功，覆盖率报告生成
