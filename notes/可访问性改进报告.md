> 招投标采集系统 Dashboard & 前端界面
> 审核时间：2026-04-07
> 审核标准：WCAG 2.1 A/AA/AAA

## 一、可访问性审计

### 1.1 色彩对比度（WCAG AA/AAA）

| 元素 | 原色值 | 对比度 | WCAG 等级 | 改进后 |
|------|--------|--------|-----------|--------|
| 主文本 (gray-900 on white) | `#111827` | 19.6:1 ✅ | AAA | 已满足 |
| 次要文本 (gray-500 on white) | `#6b7280` | 4.6:1 ⚠️ | AA 临界 | 改用 `#475569` (5.9:1) |
| 蓝色统计值 (blue-600 on white) | `#2563eb` | 4.6:1 ⚠️ | AA 临界 | 改用 `#1e40af` (8.9:1) |
| 橙色统计值 (orange-500 on white) | `#f97316` | 2.8:1 ❌ | 失败 | 改用 `#c2410c` (4.7:1) |
| 链接色 (blue-600) | `#2563eb` | 4.6:1 ⚠️ | AA 临界 | 改用 `#1d4ed8` (5.9:1) |
| SVG 图标 on 彩色背景 | `currentColor on blue-500` | ~4.5:1 ✅ | AA | 已满足 |
| 徽章背景/文字 | bg-yellow-100/text-yellow-800 | ~4.5:1 ⚠️ | AA | 需加重文字 |

**深色模式对比度问题：**
深色模式下，亮色文字在深色背景上的对比度通常良好，但 emoji 装饰符可能导致对比度不足。

### 1.2 键盘导航支持

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| ❌ 缺少跳过链接（Skip Link） | 高 | 用户无法跳过重复导航直接进入主内容 |
| ❌ 无全局 `tabindex` 管理 | 高 | 筛选按钮组使用 `@click`，键盘无法激活 |
| ❌ 筛选按钮无 `aria-pressed` | 高 | 状态型按钮必须声明按下状态 |
| ❌ 项目链接仅靠鼠标悬停提示 | 中 | `:hover` 样式键盘用户不可见 |
| ❌ 动态内容无焦点管理 | 中 | 数据更新后焦点丢失 |
| ❌ 无键盘快捷键提示 | 低 | 高级用户期望键盘操作 |

### 1.3 屏幕阅读器兼容性

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| ❌ 所有 SVG 图标无 `aria-hidden` | 高 | 屏幕阅读器会朗读空白 SVG 路径 |
| ❌ 无 `aria-live` 区域 | 高 | 数据动态更新时用户无法感知 |
| ❌ 无 `aria-label` 描述 | 高 | 链接 `查看原文 →` 无上下文 |
| ❌ 分类标题无 `aria-labelledby` | 中 | `<article>` 区块语义不明确 |
| ❌ 筛选按钮组无 `role="group"` | 中 | 屏幕阅读器无法理解按钮关系 |
| ❌ 无 `role="status"` 空状态 | 中 | 空数据提示需声明为状态消息 |
| ❌ 统计数值无 `aria-label` | 低 | `x-text="stats.total"` 动态内容需显式标注 |

### 1.4 焦点指示器

| 问题 | 严重程度 | 说明 |
|------|----------|------|
| ❌ 无自定义 `:focus-visible` 样式 | 高 | 浏览器默认蓝圈在彩色背景上不可见 |
| ❌ `outline: none` 被全局使用 | 高 | 删除所有焦点提示 |
| ❌ 卡片聚焦无视觉反馈 | 中 | `.project-card:focus-within` 存在但无默认样式 |
| ❌ 深色模式焦点色未适配 | 低 | 亮色 `outline` 在深色背景对比不足 |

## 二、包容性设计

### 2.1 深色模式支持

| 现状 | ❌ 完全缺失 |
|------|------------|

**改进方案：**
- 使用 `prefers-color-scheme: dark` 媒体查询自动切换
- CSS 变量统一管理颜色体系，亮/暗模式各一套变量
- 深色模式颜色选用符合对比度要求的色值（如 `--stat-blue: #60a5fa`）
- 手动覆盖开关（用户偏好优先于系统偏好）

### 2.2 字体大小调整

| 现状 | ❌ 无用户控制，`text-xs` (0.75rem) 过小 |
|------|----------------------------------|

**改进方案：**
- 提供小/中/大/特大 四档，通过 CSS 变量 `--font-size-base` 控制
- 使用 `rem` 而非 `px`，尊重用户浏览器默认字号
- 最小正文不小于 `0.875rem`（14px）
- 仪表盘使用 `system-ui` 字体栈，优化中文显示（Microsoft YaHei / PingFang SC）

### 2.3 动效和动画的可访问控制

| 现状 | ❌ 过渡动画无控制，`transition-shadow` 可能引发前庭功能障碍 |
|------|------------------------------------------------------------|

**改进方案：**
```css
@media (prefers-reduced-motion: reduce) {
    *, *::before, *::after {
        animation-duration: 0.01ms !important;
        transition-duration: 0.01ms !important;
    }
}
```
- 所有动画时长设为极小值（非零，避免布局抖动）
- 滚动行为 `scroll-behavior: auto`（避免平滑滚动引发问题）
- 用户手动开关：深色模式面板中提供「减少动画」切换

### 2.4 表单标签和错误提示

| 现状 | ⚠️ 本系统无传统表单，但筛选按钮存在标签问题 |
|------|---------------------------------------------|

**问题：**
- 筛选按钮计数 `<span x-text="stats.total">0</span>` 嵌套在按钮内，屏幕阅读器按 DOM 顺序读出数字，无法理解计数与按钮语义
- 错误状态（加载失败）无 `role="alert"` 通知

**改进方案：**
- 按钮使用 `aria-describedby` 引用外部计数区域
- 加载失败时用 `role="alert"` 及时通知

## 三、国际化准备

### 3.1 中文/英文双语支持架构

| 现状 | ❌ 全部硬编码中文，无 i18n 机制 |
|------|------------------------------|

**改进方案：**
- 建立 `i18n` 字典对象（已实现）：
  ```js
  const i18n = {
    zh: { page_title: "招投标采集系统仪表盘", ... },
    en: { page_title: "Tendering & Bidding Dashboard", ... }
  }
  ```
- 所有可见文本使用 `data-i18n` 属性标记
- 语言切换按钮 `aria-pressed` 状态同步
- 动态内容（数据更新、筛选变化）通过 `aria-live` 区域用当前语言通知

### 3.2 文本长度适应性

| 问题 | 说明 |
|------|------|
| 中文紧凑，英文膨胀 | "政府采购" (4字符) → "Government Procurement" (22字符) |
| 按钮宽度固定 | 中文按钮宽度不足时文字溢出 |
| Badge 标签截断 | `border-radius: 999px` + 长文本会破坏布局 |

**改进方案：**
- 布局使用 `flex` + `min-width: 0` 防止文字溢出
- 按钮内边距足够（`padding: 0.5rem 1rem`），适应文字膨胀
- Badge 使用 `max-width` 限制，超长时 `text-overflow: ellipsis`
- 卡片网格 `minmax(280px, 1fr)` 避免文字压缩
