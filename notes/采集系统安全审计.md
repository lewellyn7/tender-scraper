## 1. 输入验证与过滤 🔴 高风险

### 1.1 SSRF 漏洞 - URL 重定向未验证
**文件**: `cqggzy_crawler_v3.py`, `cqggzy_crawler_v2_fixed.py`, `ccgp_crawler_v3.py`

```python
# 问题代码
full_url = urljoin(self.BASE_URL, href)  # href 来自页面提取，未经验证
```

**风险**: 攻击者可在目标网站注入恶意链接（如 `javascript:`, `file://`, `ftp://`, `http://内部服务`），导致 SSRF。

**受影响功能**:
- 列表页链接构建（cqggzy_crawler_v3.py:84, ccgp_crawler_v3.py:96）
- 详情页附件链接构建（cqggzy_crawler_v3.py:391, ccgp_crawler_v3.py:372）
- 直接使用 `href.startswith('http')` 判断（cqggzy_crawler_v2_fixed.py:56）

**修复建议**:
```python
from urllib.parse import urlparse

def _validate_url(self, url: str) -> bool:
    """验证 URL 安全性"""
    parsed = urlparse(url)
    # 仅允许 http/https，排除协议注入
    if parsed.scheme not in ('http', 'https'):
        return False
    # 排除内网地址
    blocked = ['127.', 'localhost', '0.0.0.0', '169.254.169.254']
    return not any(parsed.netloc.startswith(b) for b in blocked)
```

### 1.2 附件链接无格式验证
**文件**: `cqggzy_crawler_v3.py:375`, `cqggzy_crawler_v2_fixed.py:275`

```python
# 问题代码 - 附件仅检查后缀
links = await page.query_selector_all('a[href$=".pdf"], a[href$=".doc"]...')
```

**风险**: 可通过双写扩展名绕过（如 `evil.pdf.exe`），且未验证 URL 协议。

**修复建议**: 使用 `mimetypes` 或额外验证文件魔数。

### 1.3 正则表达式 DoS 风险 (ReDoS)
**文件**: `cqggzy_crawler_v3.py:209`

```python
content = re.sub(r'\s+', ' ', content)  # 相对安全
```

**潜在问题**: `r'[\w\.-]+@[\w\.-]+\.\w+'` 邮箱匹配在超长输入时可能有 Catastrophic Backtracking 风险（cqggzy_crawler_v3.py:292）。

**修复建议**:
```python
email_match = re.search(r'[\w.-]+@[\w.-]+\.\w+', text, re.IGNORECASE)
```

## 2. 认证与授权机制 🟡 中风险

### 2.1 默认凭证存在于模板脚本
**文件**: `scripts/model-refresh.sh:9-10`

```bash
ADMIN_USER="${ADMIN_USER:-admin}"
ADMIN_PASS="${ADMIN_PASS:-password}"
```

**风险**: 如果环境变量未设置，使用默认密码 `password`。虽然通过环境变量覆盖，但模板本身包含不安全默认值。

**实际影响**: 该脚本用于 CI/CD 或 cron 定时任务，密码应通过 CI Secrets 注入。

**修复建议**: 脚本启动时检查环境变量是否设置，未设置则直接退出:
```bash
if [ -z "$ADMIN_PASS" ]; then
    echo "ERROR: ADMIN_PASS environment variable not set"
    exit 1
fi
```

---

### 2.2 登录接口无暴力破解防护
**文件**: API.md `/login` 端点

- 无账号锁定机制
- 无登录失败次数限制
- 无 CAPTCHA

**修复建议**: 添加 `LoginAttempt` 表记录失败次数，超过阈值后锁定 IP/账号。

### 2.3 API Key 认证无权限分级
**文件**: API.md `/api/keys`

- 创建 API Key 无权限范围限制（`models` 参数可选）

**修复建议**:

## 4. 网络安全（代理/IP防护）🔴 高风险

### 4.1 采集器无 IP 轮换机制
**文件**: 所有采集器

- 单 IP 持续请求，容易被目标网站封禁
- 无 Proxy Pool 轮换

**影响**:
- 目标网站可能 403/封禁
- 无法持续稳定采集

**修复建议**:
- 接入 Proxy Pool（参考 `API.md` 中 `/api/pool/*` 端点）
- 实现 User-Agent 轮换
- 添加请求间隔（当前 `asyncio.sleep(2)` 有一定效果）

---

### 4.2 无 User-Agent 轮换
**文件**: `chongqing_procurement_crawler.py:23-28`, `chongqing_procurement_crawler_v2.py:21-33`

```python
HEADERS = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64)...",
    # 固定 User-Agent
}
```

**风险**: 固定 User-Agent 容易被识别为爬虫。

**修复建议**: 实现 UA 池随机选择。

### 4.3 HTTPS SSL 验证开启但证书固定/自定义 CA 未配置
